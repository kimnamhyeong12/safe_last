📌 프로젝트 소개

SafeLast는 사용자가 미리 텍스트·영상·문서 형식의 유언을 작성해 보관해두고,
일정 기간 활동이 없고, 알림에 미응답하고, 서버 ping이 실패하는 등
여러 조건이 동시에 충족될 때 자동으로 유언을 가족·지인에게 전송하는 서비스입니다.

기존 “디지털 금고” 서비스와 달리,
**“사용자 행동 기반으로 사망/위험 상태를 판단하는 알고리즘”**을 적용한 것이 핵심 차별점입니다.

🧩 프로젝트 핵심 기능
1) 이메일 기반 회원가입 / 로그인

Firebase Authentication 또는 JWT 로직 기반

보안 중심의 세션 관리

2) 디지털 유언장 작성 기능

텍스트, 영상, 이미지, PDF 등 다양한 형태 지원

파일 업로드 및 서버 저장

AES 기반 이중 암호화 적용 (보안 강화)

3) 가족·지인 연락처 설정

수신자 이메일 또는 연락망 등록

전달 항목 관리 (메시지 / 파일 선택)

4) 자동 생존 체크 시스템

일정 기간 활동 없음 → 1차 의심

생존 확인 알림 미응답 → 위험 점수 증가

서버 ping 실패 → 위험 상태 신뢰도 증가

일정 점수 넘으면 유언 실행 프로세스 가동


while(true)-디지털유언장

5) 위험 상태 판단 알고리즘

PPT의 단계별 흐름 그대로 구성함:

STEP 1. 로그인 활동 없음
STEP 2. 알림 미응답
STEP 3. 서버 ping 실패
STEP 4. 위험 점수 초과 시 유언 실행

6) 사후 자동 메시지/파일 전송

위험 상태 전환 즉시 자동 발송

이메일 기반 전송

파일/메시지 해독 → 가족에게 전달

🔐 보안 구조
✔ 이중 암호화 (Double Encryption)

AES를 이용한 대칭키 암호화 (파일 암호화)

서버/클라이언트에서 복호화되기 전까지 접근 불가

전송 과정은 HTTPS 기반

✔ 서버/클라이언트 암호화 처리

작성한 유언은 서버에 암호화된 상태로 저장

유출 위험 감소
